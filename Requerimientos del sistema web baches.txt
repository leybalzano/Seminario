Requerimientos del sistema web baches.com.ar

Requerimientos Funcionales:

1)
	El sistema  deberá tener tres tipos de permisos:

 1.1	 anónimos: solo podrá ingresar y ver lo publicado en el sistema.

 1.2	 usuarios: tendrán que loguease para únicamente poder  publicar.

 1.3	 administrador: tendrán permisos totales  sobre el sistema.

2)	El sistema deberá tener un mapa con un sistema de marcadores que señalen los obstáculos en la vía pública.
 
3)	Permitir al usuario cerrar la sesión

4)	Permitirá reportar el obstáculo vía mail a la autoridad correspondiente del municipio.

5)	Transcurrido un tiempo determinado, el sistema verifica el estado del obstáculo

6)	El usuario podrán cargar la foto

7)	El usuario podrá definir por iconos los obstáculos predeterminados y categorizados/segmenteados.

8)	El usuario podrá informar el estado del obstáculo  

9)	Deberá tener un Login Screen con las siguientes opciones:

•	Registrarse: Le pedirá nombre, apellido, email y contraseña

•	Loguearse: 

•	Le pedirá usuario y contraseña, para luego ingresar al mapa
•	Podrá ingresar con su cuenta de redes sociales, para luego ir al mapa

Anonimo: Lo llevara al mapa directamente

10)	 La App debe estar disponible para su descarga en el PlayStore para Andriod y en la Tienda Window para Windows phone.

No funcional:
1)	El sistema deberá tener una interfaz simple y homogénea

2)	Se deberá acceder desde dispositivos móviles

3)	El sistema estará desarrollado a través de una plataforma web adaptable a diferentes formatos de pantalla (responsive)
